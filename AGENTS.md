# Invariant

Invariant is a distributed storage system built as a set of microservices. The system is built as a set of microservices that are connected together using HTTP.

## Rules

### Commiting changes
- After making changes, never automatically push the changes nor ask if the changes should be pushed.
- Every commit message must state that the change was generated by an LLM.
- Every commit message must include a brief explanation of the change.
- Changes should pass all tests as well as be `go fix`, `go vet`, and `go fmt` compliant.

## Implementation

### Language

The langauge the project should use is the Go programming language. The project should use the standard Go libraries as much as possible. The project should use the standard Go tools as much as possible. The project should use the standard Go build system as much as possible.

### Testing

Each module should have its own tests. The tests should be run using the standard Go test tools. The tests should be run using the standard Go build system as much as possible.

### Documentation

Each module should have its own documentation. The documentation should be run using the standard Go doc tools. The documentation should be run using the standard Go build system as much as possible.
